# Scope

## MUST

- Marketplace-neutral canonical domain.
- Wildberries as the first isolated adapter.
- Immutable raw-file storage with SHA-256.
- Schema registry and quarantine.
- Canonical event ledger.
- Configurable and versioned financial rules.
- Reconciliation.
- Evidence Chain.
- Typed missing/conflict states.
- Read-only MVP.
- Decimal arithmetic and versioned rounding.
- Idempotent import.
- Actual and Scenario separation.

## SHOULD

- Progressive onboarding.
- Exception Inbox.
- Accessibility.
- Data freshness and calculation health.
- Dead Letter Queue.
- Backup and restore verification.

## COULD

- Additional marketplace adapters.
- Optional capability modules.
- Advanced scenario analysis.

## LATER

- External write integrations.
- Production automation affecting marketplace state.

## REJECTED FOR MVP

- Kubernetes.
- Microservices without demonstrated need.
- Arbitrary user code execution.
- Hardcoded cost, tax, or other-expense constants.
