# Stage Contract A — FOUNDATION v1

## Goal

Create a verified, reproducible, and secure foundation before financial analytics
or user-facing product implementation begins.

## Units

### A0 — Autonomy readiness

- Verify tools and access.
- Recover current state.
- Resolve source of truth.
- Record blockers.

### A1 — Constitution materialization

- Preserve Constitution.
- Separate Runtime Protocol.
- Create migration notes.
- Create traceable requirements.

### A2 — GitHub bootstrap

- Dedicated repository.
- Governance structure.
- Issue Forms.
- Pull Request template.
- CODEOWNERS.
- Branch protection.
- Labels and milestones.

### A3 — Source recovery and audit

- Recover earlier artifacts.
- Check integrity.
- Classify REUSE / REVISE / REJECT / UNKNOWN.

### A4 — Data contract

- Canonical event envelope.
- Typed states.
- Provenance.
- Idempotency.
- Revision, reversal, and supersession.
- Source Authority Matrix.

### A5 — Platform foundation

- Modular-monolith skeleton.
- API and Worker entry points.
- Relational database boundary.
- Raw-file storage interface.
- Migration discipline.
- Baseline CI.

### A6 — Data proof

- One synthetic or anonymized WB file.
- Immutable raw copy.
- SHA-256.
- Structural fingerprint.
- Minimal normalization.
- Idempotency proof.
- Minimal Evidence Chain.

## Autonomous scope

R0 and R1 work inside this contract, including documentation, tests, branches,
Pull Requests, fixes, and staging-only work without commercial data.

## Gates

R3, R4, scope change, production, and external write integrations remain gated.

## Definition of Done

- GitHub is the source of truth.
- Constitution and Runtime Protocol are versioned.
- CURRENT_STATE is evidence-backed.
- Canonical event and typed-state contracts are approved.
- Raw files are hashed and immutable.
- Import is idempotent.
- Unknown schemas are quarantined.
- Evidence Chain is reproducible.
- CI, security baseline, recovery, and rollback checks pass.
- Stage A Evidence Package exists.
